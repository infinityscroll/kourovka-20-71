# An affirmative example for the `k=4` case of Kourovka Problem 20.71

## Statement answered

Problem 20.71 on printed page 154 in the 21st edition (July 3, 2026
update) of the *Kourovka Notebook* asks, for `k = 2, 3, 4`, whether a connected graph having `k`
isomorphism types among its vertex-deleted cards can have more than `k`
vertex-orbits under its automorphism group.

The graph below answers the `k = 4` case affirmatively.

## The graph

Let `G` have vertex set `{0,1,...,8}` and edge set

```text
03 04 07 14 15 16 25 26 27 28 36 37 38 48 57 68.
```

Its graph6 representation is `HCpbdgy`.  It is connected; for example,

```text
03 04 07 14 15 16 25 28
```

is a spanning tree.

## Automorphism orbits

Color refinement, initialized by degree, stabilizes at

```text
{0,5}, {1,4}, {2,3}, {6,8}, {7}.
```

Every automorphism preserves these cells.  The permutation

```text
(0 5)(1 4)(2 3)(6 8)
```

is an automorphism, so the five displayed cells are precisely the vertex
orbits.  (The exact verifier additionally enumerates all `9!` permutations
and finds only the identity and this involution.)

## Vertex-deleted cards

There are exactly four isomorphism types, with deletion vertices grouped as

```text
{0,5}, {1,4}, {2,3,6,8}, {7}.
```

The involution above supplies the isomorphisms within each of the pairs
`{0,5}`, `{1,4}`, `{2,3}`, and `{6,8}`.  Explicit maps from `G-2` to all
other members of its card class (written as old-vertex maps) are

```text
G-2 -> G-3: 0->5, 1->4, 3->2, 4->1, 5->0, 6->8, 7->7, 8->6
G-2 -> G-6: 0->2, 1->4, 3->7, 4->8, 5->1, 6->0, 7->5, 8->3
G-2 -> G-8: 0->3, 1->1, 3->7, 4->6, 5->4, 6->5, 7->0, 8->2
```

The four card types are distinct already by their degree multisets:

```text
delete 0: 2,3,3,3,3,4,4,4
delete 1: 2,2,3,3,4,4,4,4
delete 2: 2,3,3,3,3,3,3,4
delete 7: 2,2,3,3,3,3,4,4.
```

Consequently `G` has four card types but five automorphism orbits.

This matches the quantifiers literally: the graph is finite, simple,
undirected, and connected; "four types" is exactly four distinct card
isomorphism classes (with multiplicities `2,2,4,1`), not at most four.
It settles the `k=4` existence subcase only.  It does not answer the `k=2`
or `k=3` subcases.

## Elementary point verification

The orbit and card calculations above form a short certificate independent of
the discovery software: color refinement plus the displayed involution proves
that there are exactly five vertex-orbits; the displayed deletion
isomorphisms prove that there are at most four card types; and the four
different degree multisets prove that there are at least four.

`verify_kourovka_20_71_counterexample.py` uses only the Python standard
library as an exhaustive independent audit.  It determines the automorphism
group by checking all `9!` permutations and canonically classifies every card
by checking all `8!` vertex orders.

Separate point checks agree:

* NetworkX 3.3 `GraphMatcher` finds two automorphisms and the same four card
  classes.
* nauty 2.9.3 `dreadnaut` reports group size 2 and five orbits, generated by
  `(0 5)(1 4)(2 3)(6 8)`.
* nauty `delptg -l` gives the following canonical graph6 card codes, again
  yielding four types:

  ```text
  GhEIXk  GCLZLS  GaIHjo  GaIHjo  GCLZLS  GhEIXk  GaIHjo  GR`?x[  GaIHjo
  ```

These checks establish the affirmative example itself; no minimality claim is
needed for the answer to `k=4`.

`Kourovka2071.lean` gives a separate Lean 4 certificate.  It defines the
nine-vertex simple graph, proves it connected, exhaustively classifies all
automorphisms and all vertex-deleted-card isomorphisms, and concludes with

```lean
theorem affirmative_k4 :
    witness.Connected ∧
      (Finset.univ.image cardClass).card = 4 ∧
      (Finset.univ.image orbitClass).card = 5
```

It compiles in the current `formal-conjectures` environment with

```sh
lake env lean Kourovka2071.lean
```

and contains no `sorry`, `admit`, added axiom, or unsafe declaration.

## Nauty minimality census

The discovery search used `geng` and nauty certificates.  A complete census
of connected unlabeled graphs through eight vertices found no example; the
graph above was the first graph6 hit at order nine.  This minimal-order claim
depends on that exhaustive software census and is separate from the
elementary verification of the affirmative example itself.

The exact census command was

```sh
geng -cq N | python3 search_kourovka_20_71.py
```

using nauty 2.9.3 and pynauty 2.8.8.1.  The checked connected-unlabeled
graph counts were

```text
N=1: 1       N=2: 1       N=3: 2       N=4: 6
N=5: 21      N=6: 112     N=7: 853     N=8: 11117.
```

A complete continuation through all 261080 connected graphs of order nine,
using

```sh
geng -cq 9 | python3 search_kourovka_20_71.py --all-targets
```

found the displayed `k=4` witness and no `k=2` or `k=3` witness at that
order.  That latter no-hit is computational evidence about the two remaining
subcases, not a solution of them.

## Status caveat

The July 2026 Notebook still prints Problem 20.71 without a solution note,
and the separate July update file does not mention it.  The July 20, 2026
paper *On Some Problems from the Kourovka Notebook* solves eight other
problems, not 20.71.  Exact-statement searches of indexed web results, arXiv,
GitHub, and MathNet found no matching solution.

The closest historical source located was Josef Lauri's 1997 survey
[*Pseudosimilarity in Graphs---A Survey*](https://combinatorialpress.com/article/ars/Volume%20046/volume-46-paper-5.pdf),
which formulates the underlying partition-refinement question in terms of
card-equivalence versus automorphism-equivalence classes.  Its examples and
constructions do not give the four-card-type/five-orbit graph above.  Searches
of the later pseudosimilarity literature, including the 2016 second edition of
Lauri--Scapellato's *Topics in Graph Automorphisms and Reconstruction*, also
found no such example.

The graph itself is known as an unannotated data item: the exact graph6 string
`HCpbdgy` already occurs in generic Ramsey/critical-graph datasets in the
GitHub repositories `Steven-VO/circulant-Ramsey`,
`gwen-mckinley/ramsey-books-wheels`, and `melvin-adekanye/Research`.  Those
hits do not mention deletion cards, automorphism orbits, or Problem 20.71.
Thus the graph must not be claimed as new; the potentially new item is this
specific observation about it.  The negative search still does not establish
publication priority.  The indexed-literature audit found no remaining
concrete collision lead, but private correspondence and unindexed computations
remain possible.  The problem poser or the Kourovka editors should therefore
confirm priority before an unqualified novelty claim is made.
